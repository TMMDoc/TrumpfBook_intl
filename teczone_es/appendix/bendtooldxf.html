Útil de plegado DXF

Útil de plegado DXF

Flux incluye una amplia variedad de catálogos de útiles de plegado (como Trumpf, Wila, Wilson, Rolleri, Amada, etc.). Si utiliza útiles de cualquiera de los aproximadamente 40 catálogos de útiles diferentes que ofrece Flux, solo tiene que instalar el catálogo y empezar a utilizar los útiles. En ocasiones, es necesario modelar un útil personalizado o un útil que no aparece en el catálogo de Flux.El formato Bend-Tool DXF es el mecanismo estándar mediante el cual se hace esto. Flux ahora puede exportar útiles de plegado al formato DXF, y también puede importar útiles de plegado desde un formato DXF.Los archivos DXF utilizados para representar los útiles de plegado deben seguir ciertas reglas para que Flux pueda recopilar toda la información que necesita. A menudo, una forma rápida de crear un útil de plegado es utilizar la función Exportar a DXF para crear un archivo DXF a partir de un útil existente. A continuación, el archivo DXF se puede editar (ya sea en Flux o en cualquier otro sistema CAD 2D). Por último, el DXF modificado se puede volver a importar a Flux utilizando la función Importar desde DXF.

Ejemplo de DXF

A continuación se muestra un ejemplo de un útil de plegado DXF creado con el comando Exportar a DXF:

El útil DXF debe contener:

  • Una forma cerrada (formada por líneas, arcos, polilíneas) que define el perfil lateral del útil. Esta línea de perfil debe ser la POLILÍNEA cerrada más a la izquierda del dibujo (véase la figura anterior).

  • Una entidad PUNTUAL que marca el punto de referencia del útil. Esta entidad puntual debe encontrarse en algún punto dentro de la polilínea del perfil cerrado e indica el punto desde el que se mide la altura de la herramienta (véase la imagen anterior).

  • Para cada pieza de útil que no sea un simple rectángulo en la vista frontal, se requiere un dibujo de la vista frontal. Estas suelen ser piezas de abertura izquierda o derecha de la herramienta, pero también pueden ser piezas personalizadas, incluyendo piezas para ventanas.

  • Cada dibujo de la vista frontal debe aparecer a la derecha del dibujo del perfil y debe estar alineado horizontalmente con este. Cada vista delantera debe tener exactamente la misma altura que el dibujo del perfil lateral; de lo contrario, no se tendrá en cuenta. La imagen de arriba muestra un ejemplo típico con una pieza de abertura izquierda y otra de abertura derecha.

  • El resto de información sobre el útil se indica mediante cadenas de texto (véanse las notas a continuación).

Metadatos en entidades de texto

Las cadenas de texto que tienen el formato CLAVE=valor son leídas por el exportador de útiles y analizadas para obtener información adicional sobre el útil. Estas son las claves que se admiten actualmente:

NAME=OW200/K R3.2/86 CUSTOM-A
Especifica el nombre del útil. Es obligatorio.

TONNAGE=800
Especifica la carga máxima del útil, en kN/m. Esto es obligatorio si desea evitar que se muestren errores espurios de sobrecarga de útiles.

SUPERIOR=PTrumpf

Reconocimiento de formas

En el formato DXF del útil de plegado, tenga en cuenta que no es necesario especificar algunos parámetros críticos del punzón/matriz, como la altura, el radio, el ángulo o la anchura en V. Flux analiza el dibujo de la herramienta y extrae estos parámetros automáticamente. Para un punzón, Flux espera encontrar un segmento curvo en la parte inferior (bc) flanqueado por líneas rectas (ab y cd).

A partir de estos tres segmentos, se reconocen automáticamente el ángulo y el radio del punzón. Para una matriz, Flux examina el hombro izquierdo para extraer los parámetros. Espera encontrar un segmento curvo del hombro (bc) flanqueado por una parte superior horizontal (ab) y el segmento del ángulo inclinado (cd).

Si el punzón o la matriz no cumplen estos requisitos de reconocimiento de forma, Flux no podrá reconocer la forma como punzón o matriz.

Información adicional:

  • El perfil del útil puede aparecer en cualquier lugar del dibujo; la ubicación exacta no es importante, ya que la entidad del marcador de punto se utiliza para proporcionar el punto de referencia del útil.

  • Además de las entidades mencionadas anteriormente, no debe haber ninguna otra entidad en el dibujo. En particular, no debe haber ningún bloque de título ni polilíneas cerradas en el dibujo.

  • El dibujo siempre debe estar en unidades métricas (independientemente de la unidad operativa actual de Flux).

  • Aunque el comando dice Importar desde DXF, Flux también aceptará un archivo GEO, siempre que siga las mismas convenciones indicadas anteriormente.

Útiles especiales

El importador de útiles puede reconocer ciertos tipos de útiles especiales. Estos deben identificarse utilizando un texto con el formato FLAVOR=XYX en el archivo DXF.

Punzones para engatillar

Para un punzón para engatillar, se requieren las siguientes entradas adicionales:

  • El texto FLAVOR=HemPunch marca esto como una útil para engatillar.

  • El texto PARAMS=-1,12.15,70 proporciona algunas métricas que se requieren para la simulación correcta.

Hay tres parámetros necesarios para definir el proceso de engatillado. Estos se pueden ver como los valores A, B y C en la imagen de arriba. El primero (-1 mm en este ejemplo) define hasta qué punto detrás de la línea de plegado está la cara vertical plana del punzón. En este caso, dado que la cara vertical plana se encuentra delante de la línea de plegado, el valor es -1. El segundo parámetro (12,15 mm aquí) define la distancia a la que se encuentra la punta de engatillado por delante de la línea de plegado. El último parámetro (70 mm en este caso) es la distancia vertical entre la punta del punzón y la punta de engatillado.

Matrices de bloque de engatillado

Un bloque de engatillado es un accesorio que se monta delante de una matriz o detrás de una matriz para engatillado.Esto proporciona una superficie plana contra la que se puede fijar un engatillado, con el fin de realizar un engatillado limpio.

Para un bloque de engatillado, se requieren las siguientes entradas adicionales:

  • El texto FLAVOR=HemBlock marca esto como un bloque de engatillado.

  • El texto PARAMS=0,13.9 proporciona algunas métricas que se requieren para la simulación correcta.

Se requieren dos parámetros para definir el bloque de engatillado. El primer parámetro define la posición de engatillado, es decir, si este bloque debe utilizarse delante o detrás de la matriz.0 = delante y 1 = atrás. El ejemplo anterior muestra que el bloque se puede utilizar delante de la matriz.El otro parámetro es el desplazamiento del engatillado, que es el movimiento del eje I. Esto se muestra como B en la imagen de arriba.

Matrices de plegado en Z

Para una matriz de plegado en Z, debe haber una entrada de texto como esta en el archivo DXF: FLAVOR=ZBendDie

Además, el útil debe dibujarse de manera que el tramo plano central vaya de la parte inferior izquierda a la parte superior derecha. En otras palabras, la orientación debe ser como se muestra en la imagen inferior, y no la orientación invertida de esta:

Punzones de plegado en Z

Para un punzón de plegado en Z, debe haber una entrada de texto como esta en el archivo DXF: FLAVOR=ZBendPunch

Además, el útil debe dibujarse de manera que el tramo plano central vaya de la parte inferior izquierda a la parte superior derecha. En otras palabras, la orientación debe ser como se muestra en la imagen inferior, y no la orientación invertida de esta: image::ZPunch.png[,350]

Punzones de radio

Este es un eje de radio que necesita un soporte de radio.

Para tales punzones, se requieren las siguientes entradas adicionales:

  • El texto FLAVOR= RShaft marca esto como un eje de radio.

  • El texto PARAMS=80 es la altura del útil que se generará en el programa CN.

La entidad de PUNTO es el punto de referencia del útil.La herramienta debe dibujarse de modo que el punto de referencia sea donde se encuentra la punta de la V invertida del soporte.

  • Dibuje el inserto de radio con el soporte correspondiente tal y como está montado en la máquina, tal y como se muestra a continuación.

  • Retire la línea del soporte y guarde el inserto de radio dxf. Solo la entidad de PUNTO situada en la punta de la V invertida del soporte y la línea de inserto, ligeramente por debajo de PUNTO, será el perfil dxf para este inserto.

  • La altura de este útil se mide desde la entidad de PUNTO. Esto se muestra como A en la imagen de abajo.

Matrices de engatillado EV-F

Son útiles EV-F que se utilizan para realizar prensado sin desplazamiento. Este tipo de matriz se denomina I0HemDie.

Para tales matrices, se requieren las siguientes entradas adicionales:

  • El texto FLAVOR=I0HemDie marca esto como I0HemDie.

  • El texto PARAMS=8.5,95.5 proporciona algunas métricas que se requieren para la simulación correcta.

Hay dos parámetros que definen esta matriz de engatillado. Estos se pueden ver como los valores A y B en la imagen de arriba. El primer parámetro (8,5 mm en este ejemplo) define la distancia a la que se encuentra la pieza durante el engatillado desde la línea central. El segundo parámetro (95,5 mm en este caso) define la distancia vertical desde la posición de la pieza durante el engatillado hasta la parte inferior de la matriz.

Punzones para engatillar EV-F

Son punzones EV-F que se utilizan para realizar prensado sin desplazamiento. Este tipo de punzón se denomina I0HemPunch.

Para tales punzones, se requieren las siguientes entradas adicionales:

  • El texto FLAVOR=I0HemPunch marca esto como un I0HemPunch.

  • El texto PARAMS=-8,16,57.5 proporciona algunas métricas que se requieren para la simulación correcta.

Hay tres parámetros necesarios para definir el proceso de engatillado. Estos se pueden ver como los valores A, B y C en la imagen de arriba. Estos parámetros son los mismos que los definidos en HemPunch.

Matrices de plegado por rodillo

La matriz de plegado por rodillo se denomina matriz de alas.

Este útil requiere las siguientes entradas adicionales:

  • El texto FLAVOR=WingDie que marca esto como un útil de alas.

  • El texto PARAMS=10.49,100 proporciona algunas métricas que se requieren para la correcta simulación de las alas.

Hay dos parámetros que definen el movimiento de las piezas de plegado con alas. El primer parámetro es la distancia horizontal desde la línea central de la matriz hasta el punto central de rotación de una de las alas. Esto se muestra como A en la imagen de arriba. El segundo parámetro es la distancia vertical desde la parte inferior de la matriz hasta el punto central de rotación de las alas. Esto se muestra como B en la imagen de arriba. Normalmente, B y la altura serán iguales.

Matrices de autoengatillado

Son matrices en V que también tienen una mesa de engatillado separada como la imagen de abajo. Para esta matriz, debe haber una entrada de texto como esta en el archivo DXF: FLAVOR=SelfHemDie

Además, se requiere la entrada PARAMS, que tiene dos valores.Estos dos parámetros son los mismos que los de I0HemDie.

Estos se pueden ver como los valores A y B en la imagen de arriba. El primer parámetro define la distancia a la que se encuentra la pieza durante el engatillado desde la línea central. El segundo parámetro define la distancia vertical desde la posición de la pieza durante el engatillado hasta la parte inferior de la matriz.

Matrices de V doble

  • Cree dxfs para matrices con el ajuste FLAVOR=DoubleV para cada anchura de V

  • Establezca el mismo ORDERNO para ambas matrices (esta puede ser cualquier cadena no vacía)

  • Establezca el valor SHIFT en el valor IAxis correcto para usar

  • Dibuje un marcador de punto en la parte inferior de la matriz, centrado debajo de la V para indicar dónde se encuentra la geometría de conformado

Matrices de engatillado con precarga de resorte (matrices de engatillado en pluma)

Para una matriz de engatillado cargada por resorte, debe haber una entrada de texto como esta en el archivo DXF: FLAVOR=SpringHem

Además, se requiere la siguiente entrada adicional:

  • El texto PARAMS=-2,30,10 proporciona algunas métricas que se requieren para una simulación correcta.

Esta matriz es una matriz de dos piezas que tiene una parte superior que se mueve hacia abajo para aplanar el engatillado. Hay tres parámetros que definen el escote de prensado. El primer parámetro es la distancia desde la línea central de la matriz para la pieza durante el engatillado. Un valor +ve significa que está al frente de la línea central y un valor -ve significa que está detrás de la línea central. Esto se muestra como A en la imagen de arriba. El segundo parámetro es la ubicación del escote de prensado desde la base de la matriz para la pieza durante el engatillado. Esto se muestra como B en la imagen de arriba. El tercer parámetro es la ranura del resorte, que es la altura del escote. Esto se muestra como C en la imagen de arriba.

Adaptadores de punzón

El importador de útiles puede importar adaptadores de punzón. Para que Flux reconozca correctamente que el dibujo es un adaptador de punzón, debe haber las siguientes entidades de texto en el archivo:

  • Un texto como UPPER=PTrumpf que marca el tipo de montaje superior (normalmente, es PTrumpf si el adaptador se va a montar en una máquina Trumpf).

  • Un texto como LOWER=PEHT que marca el tipo de montaje inferior (es el tipo de punzón que se puede montar en el adaptador).

  • Un texto como HEIGHT=64 que mapea la altura de trabajo efectiva del adaptador. Esto es necesario, ya que Flux no puede inferir la altura del adaptador automáticamente.Aquí hay una imagen de un adaptador de punzón que se puede importar desde DXF:

Portacuchillas

El importador de útiles puede importar portacuchillas. Algunos portacuchillas (por ejemplo, los similares a OW209/S) también son punzones de engatillado utilizados para aplanar un engatillado. Para tales portacuchillas, se requieren algunos datos adicionales

Los parámetros obligatorios UPPER, LOWER, HEIGHT se explican en la sección anterior sobre adaptadores de punzón, y tienen el mismo significado. Además, para ayudar a Flux a utilizar el soporte como herramienta de engatillado, se requieren las siguientes entradas adicionales:

  • El texto SPLFLAVOR=HemPunch marca el adaptador como un útil para engatillar.

  • El texto PARAMS=-6,18,60 proporciona algunas métricas que son necesarias para la correcta simulación del proceso de engatillado.

Hay tres parámetros necesarios para definir el proceso de engatillado. Estos se pueden ver como los valores A, B y C en la imagen de arriba. El primero (-6 mm en este ejemplo) define hasta qué punto detrás de la línea de plegado está la cara vertical plana del punzón.En este caso, dado que la cara vertical plana se encuentra delante de la línea de plegado, el valor es -6. El segundo parámetro (18 mm aquí) define la distancia a la que se encuentra la punta de engatillado por delante de la línea de plegado. El último parámetro (60 mm en este caso) es la distancia vertical entre la punta del punzón y la punta de engatillado.

Soportes de radio

Este es el soporte para útiles de eje radial.

Se requieren las siguientes entradas adicionales:

  • El texto FLAVOR= RHolder marca el adaptador como un soporte de eje de radio.

  • El texto PARAMS=100 es la altura del útil que se generará en el programa CN.

  • La altura de esta herramienta se mide desde el origen hasta la V invertida del soporte.Esto se muestra como A en la imagen de abajo.

Soportes de V doble

Son sujeciones de útil inferior que pueden contener 2 matrices.

Se requieren las siguientes entradas adicionales:

  • El texto FLAVOR=DVHolder marca el adaptador como DVHolder.

  • El texto PARAMS=0,36.2 proporciona métricas para desplazar la matriz a la otra ranura en el soporte.

Hay dos parámetros necesarios para definir esta herramienta. El primero (0 mm en este ejemplo) define la distancia de la primera ranura con respecto a la referencia del soporte. En este caso, dado que la primera ranura está alineada con el punto de referencia, el valor es 0. El segundo parámetro (36,2 mm en este caso), que se muestra como A en la imagen anterior, define la distancia entre la segunda ranura y el punto de referencia del soporte.
