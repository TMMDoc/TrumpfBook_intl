ส่วนติดต่อผู้ใช้

TecZone Bend ใช้การออกแบบอินเทอร์เฟซผู้ใช้แบบใหม่ในชื่อรหัส Clarity เมื่อคุณ ทำงานใน TecZone Bend คุณจะเห็นอินเทอร์เฟซที่เรียบง่ายและไม่รกรุงรัง

ในอินเทอร์เฟซ TecZone Bend คุณจะไม่พบ:

  • แถวของไอคอนจำนวนมาก ซึ่งมีเพียงไม่กี่รายการเท่านั้นที่เกี่ยวข้องกับ บริบทปัจจุบัน

  • เมนูแบบลึกที่มีหลายสิบเมนู

  • _แถบริบบอน_ของผู้ใช้ที่ใช้พื้นที่มากด้านบนของหน้าจอ

อินเทอร์เฟซ Clarity ทำงานตามหลักการ Select แล้วจึงดำเนินการ Operate

  • เลือกสิ่งต่างๆ โดยการคลิก

  • หน้าต่าง Clarity Panel จะปรากฏขึ้น โดยมี Settings และ Operations สำหรับ สิ่งที่เลือก

  • แผงมักจะมีการควบคุมสำหรับขยายการเลือก โดยเลือกสิ่งที่คล้ายกัน

  • แผงยังมีการควบคุมสำหรับการนำทางระหว่างสิ่งต่างๆ ที่คุณกำลังทำงานด้วย

หลักการอินเทอร์เฟซผู้ใช้นี้มีความสอดคล้องกันในทุกโหมดการทำงาน TecZone Bend ไม่ว่าคุณจะจัดการโมเดล 3D, จัดเรียงชิ้นงานบนแผ่นซ้อน, หรือแก้ไขเครื่องมือเครื่องพับโลหะ คุณจะใช้ชุดแนวคิดหลักชุดเดียวกัน เพื่อทำให้แนวคิดเชิงนามธรรมเหล่านี้ชัดเจนขึ้น ลองมาดูตัวอย่างสั้นๆ ของการทำงานจริง ในอินเทอร์เฟซ Clarity (เราจะใช้ตัวอย่างจาก TecZone Bend report editor และจาก TecZone Bend เพื่ออธิบาย)

Guided Tour

นี่คือการอธิบายอย่างรวดเร็วว่าการทำงานของ Clarity เป็นอย่างไร มีแนวคิดง่ายๆ เพียงไม่กี่ข้อที่ถูกใช้ซ้ำทุกที่ใน TecZone Bend และการเข้าใจ แนวคิดเหล่านี้ทำให้สามารถใช้ TecZone Bend ส่วนใหญ่ได้โดยไม่ต้องอ่านคู่มือ หรือฝึกอบรมมากนัก กล่าวอีกอย่างคือ หากคุณอ่านส่วนนี้ คุณสามารถข้ามการอ่าน คู่มือผู้ใช้ TecZone Bend ที่เหลือได้!

ตัวอย่างรายงาน

คุณสามารถเรียนรู้เพิ่มเติมเกี่ยวกับการสร้างและแก้ไขแม่แบบรายงานในส่วนถัดไป ตอนนี้ขอแนะนำศัพท์เฉพาะของรายงานบางคำ เพื่อใช้ใน การอธิบายแนวคิดของอินเทอร์เฟซ Clarity

ภาพด้านบนแสดงประเภทของ entities ที่สามารถวางลงในรายงานได้

  • Label ประเภทหนึ่งคือข้อความธรรมดาที่สามารถตั้งค่าฟอนต์ การจัดวางข้อความ และสีได้

  • Field จะแสดงข้อมูลจริงบางส่วนที่ดึงมาจากชิ้นงานที่เป็นพื้นฐาน

  • Shape คือรูปทรงเรขาคณิตง่ายๆ (เช่น วงรี สี่เหลี่ยม หรือเส้น) ที่สามารถ ใช้เป็นพื้นหลัง ไฮไลท์ หรือเส้นแบ่งในรายงานได้

  • Table จะแสดงข้อมูลจากชิ้นงานในรูปแบบตาราง (คล้ายกับสเปรดชีตของข้อมูล) ในตัวอย่างนี้ เราจะเห็นรายการการดัดของชิ้นงาน และมีการแสดงคุณลักษณะสำหรับแต่ละรายการ

ด้วยพื้นฐานเกี่ยวกับ entities ของรายงานนี้ เราสามารถเริ่มสำรวจอินเทอร์เฟซ Clarity ได้แล้ว

หลักการ: คลิกที่ใดที่หนึ่งเพื่อแก้ไข

เมื่อคลิกที่ Field และ Clarity Panel จะปรากฏขึ้น (ในที่นี้ จะเรียกว่า Panel) แผนภาพที่มีคำอธิบายแสดงให้เห็นว่าหน้าตาจะเป็นอย่างไร:

การคลิกที่ฟิลด์จะเลือกฟิลด์นั้นและแสดง Panel สำหรับแก้ไของค์ประกอบบางส่วนที่เราเห็นในแผงนี้ ได้แก่:

  • แถบชื่อใช้ย้ายตำแหน่งของแผง (สามารถ คลิกที่พื้นหลังของแผง แล้วลาก) หากคุณปักหมุด ไว้ แผงจะเปิดขึ้นที่ตำแหน่งนั้นบนหน้าจอเสมอ หากไม่ปักหมุด แผงจะเปิดใกล้ตำแหน่งของเคอร์เซอร์เมาส์

  • กล่องแก้ไข กล่องป้อนตัวเลข รายการตัวเลือก ตัวเลือกฟอนต์ และการจัดตำแหน่ง เป็นวิธีต่างๆ ที่ใช้ดูและแก้ไขการตั้งค่าของฟิลด์

  • ปุ่มการทำงานใช้สำหรับดำเนินการกับวัตถุที่เลือก ในกรณีนี้ สามารถลบฟิลด์ออกจากรายงานได้โดยคลิก ปุ่ม Delete

  • แผงบางแผงอาจมีองค์ประกอบจำนวนมาก เพื่อลดความรก แผงเหล่านี้ ถูกแบ่งออกเป็นส่วนๆ โดยใช้ Expanders ในภาพด้านบน คุณสามารถเห็น expander Visual หนึ่งที่ถูกปิด (ซ่อนการตั้งค่าทั้งหมดในส่วนนั้น) ในขณะที่ expander ของ Font ถูกเปิด สามารถคลิกที่ expander ใดก็ได้ เพื่อเปิดหรือปิดส่วนนั้น

หลักการ: เลือกหลายสิ่งและแก้ไขพร้อมกัน

สามารถเลือกเอนทิตีเพิ่มเติมได้โดยกดปุ่ม Shift ค้างไว้ เมื่อคลิก เมื่อเลือกเอนทิตีหลายรายการ แผงหนึ่งจะแสดงขึ้น เพื่อให้แก้ไขการตั้งค่าของทั้งหมดได้พร้อมกัน

ในภาพด้านบน เราเลือกฟิลด์ 3 ฟิลด์โดยใช้กลไก Shift+Click แผงที่แสดงขึ้นอนุญาตให้แก้ไขการตั้งค่าของทั้งสามฟิลด์ ได้พร้อมกัน (ในที่นี้ expander Visual หนึ่งถูกเปิด และอีก expander Font หนึ่งถูกปิด) หลังจากแก้ไขการตั้งค่าสี Bgrd และการตั้งค่า Pen-width รายงานจะมีลักษณะดังนี้:

การเปลี่ยนแปลงทั้งหมดถูกนำไปใช้กับรายการที่เลือก ในสถานการณ์การแก้ไข หลายรายการ โปรดสังเกตว่าการตั้งค่า Expression จะไม่ปรากฏ ซึ่งเป็น การตั้งค่าที่ไม่เหมาะสมในการแก้ไขพร้อมกัน (เช่น ไม่จำเป็นต้องมีฟิลด์ 3 ฟิลด์ที่แสดงค่าจาก Part เดียวกัน) การตั้งค่าที่ปกติจะมีค่าแตกต่างกันสำหรับแต่ละเอนทิตีจะไม่ปรากฏในแผงแก้ไขกลุ่ม

หลักการ: สามารถแก้ไขเอนทิตีต่างชนิดกันพร้อมกันได้

ในตัวอย่างด้านบน เราเลือกเอนทิตี 3 รายการที่เป็นชนิดเดียวกัน แล้วถ้าเลือก เอนทิตีต่างชนิดกันจะเป็นอย่างไร มาดูกัน นี่คือลักษณะของแผง เมื่อเลือกเอนทิตี Shape:

หากเลือก Field และ Shape พร้อมกัน แผงนี้จะปรากฏขึ้น:

  • ตอนนี้ แถบชื่อจะแสดงข้อความว่า 2 Report Items เนื่องจากเรากำลังแก้ไขรายการรายงานสองรายการ แต่ไม่ใช่ชนิดเดียวกัน

  • เฉพาะการตั้งค่าที่เหมือนกันระหว่าง Field กับ Shape เท่านั้นที่จะปรากฏขึ้น (ในที่นี้คือการตั้งค่าในส่วน Visual)

หลักการ: การเลือกหลายรายการอาจทำให้มีการดำเนินการเพิ่มเติมได้

นอกเหนือจากชุดการตั้งค่าและการทำงานที่ใช้ได้ทั่วไปสำหรับหลายวัตถุ การเลือกหลายรายการอาจแสดงการทำงานหรือการตั้งค่าใหม่ ที่ไม่สามารถใช้ได้กับวัตถุเดี่ยว มาลองดูตัวอย่างจาก มุมมองการดัดเพื่ออธิบายสิ่งนี้ ในมุมมองการดัด การคลิกที่การดัดหนึ่งจะแสดง แผงสำหรับแก้ไขการตั้งค่าของการดัดนั้นและดำเนินการบางอย่าง:

ภาพด้านบนแสดงแผงสำหรับการดัดหมายเลข 3 ตอนนี้ให้เรากด Shift ค้างไว้ แล้วเลือกการดัดหมายเลข 4 เพิ่มด้วย แผงที่แสดงตอนนี้มีลักษณะดังนี้:

มีการตั้งค่าบางอย่างที่สามารถแก้ไขร่วมกันได้ (เช่น การตั้งค่าข้ามการดัด หรือการตั้งค่าหมายเลขสถานี) อย่างไรก็ตาม เราเห็นปุ่มใหม่ปรากฏขึ้นที่นี่ชื่อว่า Group เนื่องจากมีการเลือกการดัดสองรายการและสองรายการนี้อยู่ในแนวเส้นตรงเดียวกัน จึงสามารถจัดกลุ่มเป็นการดัดเดียวได้ ซึ่งจะตรวจจับสิ่งนี้โดยอัตโนมัติและ จะแสดงปุ่ม Group ใหม่นี้ขึ้นมา การทำงานนี้จะไม่ปรากฏ หากเลือกเพียงการดัดเดียว การคลิกที่ปุ่มนั้นจะจัดกลุ่มการดัดทั้งสองเข้าด้วยกัน ดังที่เห็นในภาพด้านล่าง

ตอนนี้มีการดัดเพียง 3 รายการในชิ้นงานนี้ นอกจากนี้ การดัดหมายเลข 3 ยังถูก จัดกลุ่ม และคุณจะเห็นปุ่มใหม่ปรากฏขึ้น: Ungroup

หลักการ: ใช้แผงย่อยการเลือกเพื่อปรับเปลี่ยนการเลือก

แผง Clarity มักจะมีแผงย่อย Select อยู่ใกล้ด้านล่าง แผงนี้จะมีปุ่มบางอย่างที่ช่วยปรับเปลี่ยนการเลือกในรูปแบบต่างๆ โดยปกติจะมีปุ่มสำหรับเลือกวัตถุอื่นๆ ที่คล้ายกันไม่ทางใดก็ทางหนึ่ง ตัวอย่างเช่น วิธีการขยายการเลือก ได้แก่:

  • เลือกฟิลด์ทั้งหมดในรายงาน

  • เลือกเส้นโพลีไลน์ทั้งหมดในเลเยอร์เดียวกัน

  • เลือกตำแหน่งการเจาะทั้งหมดที่ใช้หัวเจาะเดียวกัน

  • เลือกการดัดทั้งหมดที่ใช้การตั้งค่าเดียวกัน

นี่คือตัวอย่างจากตัวแก้ไขรายงานของแผงการเลือกที่ถูกแสดงขึ้น เมื่อเราคลิกที่ฟิลด์ในรายงาน:

แผงย่อยการเลือกนี้มีพื้นหลังลายเฉพาะและปุ่มทรงกลม ในตัวอย่างนี้ คุณจะเห็นปุ่มสำหรับเลือก All Fields เลือก All Bend Fields (ฟิลด์ที่เกี่ยวข้องกับข้อมูล CAM ของการดัด) และ Toggle (ชุดการเลือกปัจจุบัน)

รายการปุ่มการเลือกที่แสดงในแผงย่อยนี้จริง จะเปลี่ยนไปตามบริบท คุณจะพบว่าสถานการณ์ทั่วไปส่วนใหญ่สามารถจัดการได้ อย่างสะดวกโดยใช้ปุ่มนี้

หลักการ: ใช้แผงย่อยการนำทางเพื่อเดินตามโครงสร้างเอนทิตี

เซตของเอนทิตีในชิ้นงาน รายงาน หรือเลย์เอาต์ มักจะมีโครงสร้างลำดับชั้น เราเรียกสิ่งนี้ว่า Entity Tree โครงสร้างนี้มักจะแสดงเป็นต้นไม้กลับหัว โดยมีวัตถุ "แม่" (parent) อยู่ด้านบนและ "ลูก" (children) อยู่ด้านล่าง ตัวอย่างเช่น นี่คือต้นไม้ของเอนทิตีในรายงานหนึ่ง

  • รายงานหนึ่งสามารถมีหลายรายการอยู่ภายใน บางรายการอาจเป็นตาราง

  • ตารางหนึ่งในรายงานอาจมีหลายคอลัมน์

ดังนั้น ต้นไม้นี้จึงมี 3 ระดับ และสามารถแสดงภาพ เป็นกราฟิกดังนี้:

รายงานนี้มี 2 Labels, 1 Field และ 1 Table ตารางนั้น 4 คอลัมน์ เมื่อเราคลิกที่ตาราง จะเห็นแผงดังนี้:

แผงการนำทางจะมีปุ่มนำทางที่ช่วยให้เดินไปตามโครงสร้างต้นไม้ ของเอนทิตีได้ เช่นเดียวกับแผงการเลือก แผงย่อยการนำทางมีพื้นหลังแบบลายเฉพาะ และปรากฏที่ด้านล่างของแผง แต่จะอยู่เหนือแผงย่อยการเลือกเล็กน้อย

ปุ่ม Prev และ Next ใช้สำหรับเลื่อนไประหว่างรายการ ในระดับเดียวกันของต้นไม้เอนทิตี ดังนั้น การคลิกปุ่มเหล่านี้จะเป็นการเลือกเอนทิตีอื่น ๆ ในรายงาน เช่น Label หรือ Field ในรายงาน

การคลิกปุ่ม Report จะเลื่อนการเลือกจากตารางขึ้นไปยังวัตถุแม่ คือ Report เอง แผงที่แสดงขึ้นเมื่อคุณคลิก ปุ่ม Report จะแสดงการตั้งค่าบางอย่างที่เกี่ยวข้องกับรายงานเอง เช่น ขนาดกระดาษและระยะขอบ:

ตอนนี้เรากำลังแก้ไขรายงาน เราอยู่ที่รากของโครงสร้างเอนทิตี ไม่มีทิศทาง ให้ไปต่อได้นอกจากลง ดังนั้นแผงย่อยการนำทางจึงมีเพียง ปุ่มเดียว (รายการ) เพื่อเลื่อนไปยังรายการของเอนทิตี การคลิกปุ่มนี้ จะเลือกตารางอีกครั้ง

จากตาราง ปุ่มนำทางลง Columns จะเลื่อนเราลงไปอีกหนึ่งระดับ และเข้าสู่โหมดแก้ไขคอลัมน์ นี่คือสิ่งที่เกิดขึ้นเมื่อเราคลิก ปุ่มนำทาง Columns จากแผงตาราง:

การตั้งค่าสำหรับคอลัมน์ที่เลือกในตารางจะปรากฏขึ้น การคลิกปุ่ม Prev และ Next จะเลื่อนการเลือกไปมา ระหว่าง_คอลัมน์_ในตาราง เนื่องจากนี่คือระดับในโครงสร้างเอนทิตีที่เรากำลัง ทำงานอยู่ ไม่มีปุ่มนำทางลงอีกแล้ว เนื่องจากนี่คือระดับต่ำสุดใน โครงสร้างเอนทิตี ปุ่ม Table จะเลื่อนขึ้นไปหนึ่งระดับและเรา กลับไปที่การแก้ไขตารางทั้งหมดอีกครั้ง
