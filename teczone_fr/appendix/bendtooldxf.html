Outil de pliage DXF

Outil de pliage DXF

Flux est livré avec une large gamme de catalogues des outils de pliage (comme TRUMPF, Wila, Wilson, Rolleri, Amada, etc.). Si vous utilisez des outils provenant de l’un des quelque 40 catalogues des outils mis à disposition par Flux, il vous suffit d’installer le catalogue et de commencer à utiliser les outils. Il est parfois nécessaire de modéliser un outil personnalisé ou un outil provenant d’un catalogue non représenté dans Flux.

Le format DXF de l’outil de pliage est le mécanisme standard utilisé à cet effet. Flux peut désormais exporter des outils de pliage au format DXF et importer des outils de pliage à partir d’un format DXF. Les fichiers DXF utilisés pour représenter les outils de pliage doivent respecter certaines règles afin que Flux puisse recueillir toutes les informations dont il a besoin. Très souvent, un moyen rapide de créer un outil de pliage consiste à utiliser la fonction Exporter vers DXF pour créer un fichier DXF à partir d’un outil existant. Ensuite, le fichier DXF peut être modifié (soit dans Flux, soit dans n’importe quel autre système CAO 2D). Enfin, le fichier DXF modifié peut être réimporté dans Flux à l’aide de la fonction Importer depuis DXF.

Chaque vue de face doit avoir exactement la même hauteur que le plan du profilé latéral, sinon elle ne sera pas prise en considération. Détection de forme

Dans le format DXF de l’outil de pliage, veuillez noter qu’il n’est pas nécessaire de spécifier certains paramètres critiques du poinçon/de la matrice tels que la hauteur, le rayon, l’angle ou la largeur en V. Flux analyse le dessin de l’outil et extrait ces paramètres automatiquement. Pour un poinçon, Flux s’attend à trouver un segment courbe dans la partie inférieure (bc) flanqué de lignes droites (ab et cd).

A partir de ces trois segments, l’angle et le rayon du poinçon sont automatiquement reconnus. Pour une matrice, Flux examine l’épaulement gauche pour extraire les paramètres. Il s’attend à trouver un segment d’épaulement incurvé (bc) flanqué d’un sommet horizontal (ab) et d’un segment d’angle incliné (cd).

Si le poinçon ou la matrice ne répondent pas à ces exigences de reconnaissance de forme, Flux ne sera pas en mesure de reconnaître la forme comme un poinçon ou une matrice.

Informations additionnelles :

  • Le profilé de l’outil peut apparaître n’importe où dans le dessin – l’emplacement précis n’a pas d’importance puisque l’entité de marquage de point sert à fournir le point de référence de l’outil.

  • Outre les entités mentionnées ci-dessus, aucune autre entité ne doit figurer dans le dessin. En particulier, il ne doit pas y avoir de bloc-titre ni de polylignes fermées dans le dessin.

  • Le dessin doit toujours être en unités métriques (quelle que soit l’unité de fonctionnement actuelle de Flux).

  • Bien que l’ordre indique Importer depuis DXF, Flux acceptera également un fichier GEO, à condition qu’il respecte les conventions énumérées ci-dessus.

Outils spéciaux

L’importateur d’outils peut reconnaître certains types d'outils spéciaux. Ceux-ci doivent être identifiés à l’aide d’un texte sous la forme FLAVOR=XYX dans le fichier DXF.

Poinçons de réalisation de plis écrasés

Pour un poinçon de réalisation de plis écrasés, les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=HemPunch indique qu’il s’agit d’un outil type pli écrasé.

  • Le texte PARAMS=-1,12.15,70 fournit certaines mesures nécessaires à une simulation correcte.

Trois paramètres sont nécessaires pour définir le processus de réalisation de plis écrasés. Ces valeurs correspondent aux valeurs A, B et C dans l’image ci-dessus. Le premier (-1 mm dans cet exemple) définit la distance entre la ligne de pliage et la face verticale plane du poinçon. Dans ce cas, comme la face verticale plane se trouve devant la ligne de pliage, la valeur est -1. Le deuxième paramètre (12,15 mm ici) définit la distance entre la pointe de réalisation de plis écrasés et la ligne de pliage. Le dernier paramètre (70 mm ici) correspond à la distance verticale entre la pointe du poinçon et la pointe de réalisation de plis écrasés.

Matrices 4 vés de réalisation de plis écrasés

Un bloc de réalisation de plis écrasés est une pièce annexe qui est montée devant une matrice ou derrière une matrice pour la réalisation de plis écrasés. Cela permet d’obtenir une surface plane contre laquelle un écrasement peut s’engager, afin de réaliser un pli écrasé net.

Pour un bloc de réalisation de plis écrasés, les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=HemBlock indique qu’il s’agit d’un bloc de réalisation de plis écrasés.

  • Le texte PARAMS=0,13.9 fournit certaines mesures nécessaires à une simulation correcte.

Deux paramètres sont nécessaires pour définir le bloc de réalisation de plis écrasés. Le premier paramètre définit la position d’écrasement, c’est-à-dire si ce bloc doit être utilisé devant ou derrière la matrice. 0 = devant et 1 = derrière. L’exemple ci-dessus montre que le bloc peut être utilisé devant la matrice. L’autre paramètre est la déviation de l’écrasement qui est le mouvement d’axe I. Ceci est représenté par l’élément B dans l’image ci-dessus.

Poinçons radiaux

Il s’agit d’un arbre radial qui nécessite un support radial.

Pour ces poinçons, les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=RShaft indique qu’il s’agit d’un arbre radial.

  • Le texte PARAMS=80 correspond à la hauteur de l’outil à éditer dans le programme CN.

L’entité de POINT est le point de référence de l’outil. L’outil doit être dessiné de manière à ce que le point de référence se trouve à l’endroit où se trouve la pointe du V inversé du support de base.

  • Dessinez l’insert radial avec le support correspondant tel qu’il est monté dans la machine, comme illustré ci-dessous.

  • Retirez la ligne de support et enregistrez l’insert radial DXF. Seule l’entité de POINT située à l’extrémité du V inversé du support de base et la ligne d’insert, légèrement en dessous du POINT, constitueront le profilé DXF de cet insert.

Matrices de réalisation de plis écrasés EV-F

Il s’agit d’outils EV-F utilisés pour la réalisation de plis écrasés sans déviation. Une telle matrice est appelée I0HemDie.

Pour ces matrices, les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=I0HemDie indique qu’il s’agit d’une I0HemDie.

  • Le texte PARAMS=8.5,95.5 fournit certaines mesures nécessaires à une simulation correcte.

Deux paramètres définissent cette matrice de réalisation de plis écrasés. Ces valeurs correspondent aux valeurs A, et B dans l’image ci-dessus. Le premier paramètre (8,5 mm dans cet exemple) définit la distance entre la pièce et la ligne centrale pendant la réalisation de plis écrasés. Le deuxième paramètre (95,5 mm ici) définit la distance verticale entre l’emplacement de la pièce pendant la réalisation de plis écrasés et la partie inférieure de la matrice.

Poinçons de réalisation de plis écrasés EV-F

Il s’agit de poinçons EV-F utilisés pour la réalisation de plis écrasés sans déviation. Un tel poinçon est appelé I0HemPunch.

Pour ces poinçons, les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=I0HemPunch qui indique qu’il s’agit d’un I0HemPunch.

  • Le texte PARAMS=-8,16,57.5 fournit certaines mesures nécessaires à une simulation correcte.

Trois paramètres sont nécessaires pour définir le processus de réalisation de plis écrasés. Ces valeurs correspondent aux valeurs A, B et C dans l’image ci-dessus. Ces paramètres sont identiques à ceux définis dans HemPunch.

Matrices de cintrage

La matrice de cintrage est appelée matrice à ailes.

Cet outil nécessite les saisies supplémentaires suivantes :

  • Le texte FLAVOR=WingDie qui indique qu’il s’agit d’un outil à ailes.

  • Le texte PARAMS=10.49,100 fournit certaines mesures nécessaires à une simulation correcte des ailes.

Il y a deux paramètres définissant le mouvement des pièces de pliage à ailes. Le premier paramètre est la distance horizontale entre la ligne médiane de la matrice et la pointe de centrage de rotation de l’une des ailes. Ceci est représenté par l’élément A dans l’image ci-dessus. Le deuxième paramètre est la distance verticale entre la partie inférieure de la matrice et la pointe de centrage de rotation des ailes. Ceci est représenté par l’élément B dans l’image ci-dessus. Normalement, B et Hauteur sont identiques.

Ces valeurs correspondent aux valeurs A, et B dans l’image ci-dessus. Le premier paramètre définit la distance entre la pièce et la ligne centrale pendant la réalisation de plis écrasés. Le deuxième paramètre définit la distance verticale entre l’emplacement de la pièce pendant la réalisation de plis écrasés et la partie inférieure de la matrice.

Matrices à 2 V

Les matrices qui ont plusieurs largeurs de V et qui nécessitent IAxis pour utiliser l’autre V sont appelées matrices à 2 V. Il y a quelques règles à respecter :

  • Créez des fichiers DXF pour les matrices avec le paramètre FLAVOR=DoubleV défini pour chaque largeur V

  • Définissez le même ORDERNO pour les deux matrices (il peut s’agir de n’importe quelle chaîne non vide)

  • Réglez la valeur YSHIFT sur la valeur IAxis correcte à utiliser

  • Tracez un repère au niveau de la partie inférieure de la matrice, centré sous le V, pour indiquer l’emplacement de la géométrie de déformation

Adaptateurs poinçon

L’importateur d’outils peut importer des adaptateurs poinçon. Pour que Flux reconnaisse correctement que le dessin est un adaptateur poinçon, les entités de texte suivantes doivent figurer dans le fichier :

  • Un texte tel que UPPER=PTrumpf indiquant le type de montage supérieur (il s’agit généralement de PTrumpf si l’adaptateur doit être monté sur une machine TRUMPF).

  • Un texte tel que LOWER=PEHT indiquant le type de montage inférieur (il s’agit du type de poinçon pouvant être monté sur l’adaptateur).

  • Un texte tel que HEIGHT=64 qui correspond à la hauteur de travail effective de l’adaptateur. Ceci est nécessaire, car Flux ne peut pas déduire automatiquement la hauteur de l’adaptateur.

Supports de lame

L’importateur d’outils peut importer des supports de lame. Certains supports de lame (par exemple, ceux similaires à OW209/S) sont également des poinçons de réalisation de plis écrasés utilisés pour aplatir un ourlet. Pour ces supports de lame, certaines données supplémentaires sont requises

Les paramètres obligatoires UPPER, LOWER, HEIGHT sont expliqués dans la section ci-dessus sur les adaptateurs poinçon et ont la même signification. De plus, pour aider Flux à utiliser le support comme outil de réalisation de plis écrasés, les entrées supplémentaires suivantes sont requises :

  • Le texte SPLFLAVOR=HemPunch indique que l’adaptateur est un outil de réalisation de plis écrasés.

  • Le texte PARAMS=-6,18,60 fournit certaines mesures nécessaires à une simulation correcte du processus de réalisation de plis écrasés.

Trois paramètres sont nécessaires pour définir le processus de réalisation de plis écrasés. Ces valeurs correspondent aux valeurs A, B et C dans l’image ci-dessus. Le premier (-6 mm dans cet exemple) définit la distance entre la ligne de pliage et la face verticale plane du poinçon. Dans ce cas, comme la face verticale plane se trouve devant la ligne de pliage, la valeur est -6. Le deuxième paramètre (18 mm ici) définit la distance entre la pointe de réalisation de plis écrasés et la ligne de pliage. Le dernier paramètre (60 mm ici) correspond à la distance verticale entre la pointe du poinçon et la pointe de réalisation de plis écrasés.

Supports radiaux

Il s’agit du support pour outils à arbre radial.

Les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=RHolder indique que l’adaptateur est un support radial.

  • Le texte PARAMS=100 correspond à la hauteur de l’outil à éditer dans le programme CN.

  • La hauteur de cet outil est mesurée depuis l’origine jusqu’au V inversé du support. Ceci est représenté par A dans l’image ci-dessous.

Supports pour matrices 2 V

Il s’agit de supports matrice pouvant accueillir 2 matrices.

Les saisies supplémentaires suivantes sont requises :

  • Le texte FLAVOR=DVHolder indique que l’adaptateur est un DVHolder.

  • Le texte PARAMS=0,36.2 fournit des mesures permettant de déplacer la matrice vers l’autre rainure dans le support.

Deux paramètres sont nécessaires pour définir cet outil. Le premier (0 mm dans cet exemple) définit la distance entre la première rainure et la référence du support. Dans ce cas, comme la première rainure est alignée sur le point de référence, la valeur est 0. Le deuxième paramètre (36,2 mm ici), indiqué par A dans l’image ci-dessus, définit la distance entre la deuxième rainure et le point de référence du support.
