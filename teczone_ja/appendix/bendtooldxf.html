曲げツール DXF

Flux には、さまざまな曲げツール カタログ (Trumpf、Wila、Wilson、Rolleri、Amada など) が付属しています。Flux の約 40 種類のツール カタログのいずれかのツールを使用している場合は、カタログをインストールするだけでツールを使用開始できます。時には、カスタムツールや、フラックス にないカタログのツールをモデリングする必要があります。 Tool DXFフォーマット は、これを行うための規格のメカニズムです。フラックスは曲げツールをDXFフォーマットにエクスポートしたり、DXFフォーマットから曲げツールをインポートできます。曲げツールを表現するために使用される DXF ファイルは、フラックス が必要なすべての情報を取得できるように、特定のルールに従う必要があります。多くの場合、曲げツールを素早く作成するため、Export to DXF機能を使って既存のツールからDXFファイルを作成します。次に、DXFを（ フラックス、または他の2 D CADシステムのいずれかで）編集します。最後に、「DXF からインポート」機能を使用して 、修正された DXF をフラックスに戻します。

ツールDXFには以下が必要です：

  • ツールの_サイドプロファイル_を規定する1つの閉じた形状（線、円弧、ポリラインからなる）。このプロファイルプラインは、図面で最も左端にある閉じたポリラインでなければなりません （上の図を参照）。

  • ツール原点をマーキングする点エンティティ。この点エンティティは 閉じたプロファイルポリライン内でなくてはならず、ツールの高さを測定する点です (上の図を参照)。

  • 正面図が単純な長角ではないツールピースには、_正面図の描画_が必要です。これらは通常、左または右イヤーピースですが、_ウィンドウ_ピースなどのカスタム ピースのこともあります。

  • 各正面図描画をプロファイル描画の右、そして 水平方向に揃えます。各正面図は、側面プロファイル図と正確に同じ高さでなければなりません；そうでない場合、その正面図は認識されません。上の画像は、左右イヤピースの典型的な例です。

  • ツールに関するその他の情報は、テキスト文字列に表示されます（下の注記を参照）。

    はツール名称を特定します。これは必須です。

形状認識

曲げツールツールDXFフォーマットでは、高さ、半径、角度、またはV幅などのパンチ/ダイのいくつかの重要なパラメーターを特定する必要がありません。フラックスはツール描画を分析し、これらのパラメータを自動的に抽出します。パンチの場合、フラックス は、直線 (ab と cd) に囲まれた下部の曲線セグメント (bc) を想定します。

これらの3つのセグメントから、パンチの角度と半径を自動的に認識します。ダイの場合、フラックスは左肩を調べてパラメータを抽出します。水平な上面 (ab) と傾斜した角度のセグメント (cd) に挟まれた湾曲した肩のセグメント (bc) を想定します。

パンチやダイがこれら形状認識要件を満たさない場合、フラックスは形状をパンチやダイとして認識できません。

追加情報：

  • ツール プロファイルは描画のどこかに表示されます　- 正確な場所は重要ではありません。ポイント マーカー エンティティでツールの原点がわかるからです。

  • 上記のエンティティ以外には、描画には他のエンティティがあってはなりません。特に、描画にはタイトル ブロックや閉じたポリラインがあってはなりません。

  • 描画は常にメートル単位にして下さい（フラックスの現在のオペレートユニットに関わらず）。

  • コマンドは DXF からインポートするようになっていますが、フラックスは、上記の規定に従っている限り、GEO ファイルも受け入れます。

特殊ツール

ツールインポーターは、あるタイプの_特殊ツール_は認識できます。これらは、DXF ファイルの FLAVOR=XYX 形式のテキストを使用して識別されます。

完全折り畳み用パンチ

完全折り畳み用パンチには、以下も入力して下さい：

  • テキスト FLAVOR = HemPunch はこれが完全折り畳みツールであることを示します。

  • テキスト PARAMS =-1,12.15,70 は、正しいシミュレーションに必要ないくつかの指標を提供します。

完全折り畳み加工プロセスを規定するには、3つのパラメーターが必要です。つまり、上の画像のA、B、Cの値です。最初の値 (この例では -1mm) は、パンチの平らな垂直面が曲げ線からどれだけ後ろにあるかを規定します。この場合、垂直な平面は曲げ線の前なので、値は-1です。2番目のパラメーター（ここでは12.15 mm ）は、完全折り畳みの先端が曲げ線のどれだけ前方になるかを規定します。最後のパラメーター（ここでは70 mm ）は、パンチの先端から完全折り畳みの先端までの垂直距離です。

完全折り畳みブロックダイ

完全折り畳みブロックは、完全折り畳みのために、ダイの前、または後ろ用の取付けパーツです。これは平らな面で、完全折り畳みのきれいな折り返しのために使います。

完全折り畳みブロックには、さらに以下の入力が必要です：

  • テキスト FLAVOR = HemBlock はこれが完全折り畳みブロックなことを示します。

  • テキスト PARAMS = 0,13.9 は、正しいシミュレーションに必要ないくつかの指標を提供します。

完全折り畳みブロックの規定には、2つのパラメーターが必要です。最初のパラメーターは、このブロックをダイの前または後ろで使用するかという完全折り畳み位置を規定します。0 =前面、1 =背面。上の例は、ブロックをダイの前で使用できることを示します。もう1つのパラメータは、I軸の動きでヘミング偏向です。これは上の画像の B です。

Z曲げ加工ダイ

Z曲げ加工の場合、DXFファイルに以下のテキスト入力があるはずです： FLAVOR = ZBendDie

さらに、ツールは中央の平坦な部分が左下端から右上端に向かって描かれるように作図して下さい。言い換えれば、ツールの向きは下の画像のようにする必要があり、これを_反転させた_向きにはしないでください。

Z曲げ加工パンチ

Z曲げ加工パンチには、DXFファイルに次のようなテキスト入力が必要です： FLAVOR = ZBendPunch

さらに、ツールは中央の平坦な部分が左下端から右上端に向かって描かれるように作図して下さい。言い換えれば、ツールの向きは下の画像のようにする必要があり、これを_反転させた_向きにはしないでください。

半径パンチ

これは半径ベースサポートが要る半径シャフトです。

このようなパンチには、さらに以下の入力が必要です。

  • テキスト FLAVOR = RShaft はこれが半径シャフトなことを示します。

  • テキスト PARAMS = 80 は、NCプログラムが出力するツールの高さです。

ポイントエンティティは、ツール原点です。ツールは、ホルダーの逆 V の先端が原点になるように描画して下さい。

  • ホルダープラインを取り外し、半径アライメントdxfを保存します。ホルダーのインバート V先端のPOINT エンティティと、POINT から少し下にあるアライメントインサートプラインのみが、このアライメントのプロファイル DXF になります。

EV - F　完全折り畳みダイ

これらは、偏向なしの完全折り畳みに使用されるEV - Fツールです。そのようなダイは、I0HemDie と呼ばれます。

このようなダイには、次の入力が必要です。

  • テキスト FLAVOR = I 0 HemDie はこれがI 0 HemDieなことを示します。

  • テキスト PARAMS = 8.5,95.5 は、正しいシミュレーションに必要ないくつかの指標を提供します。

2つのパラメーターで完全折り畳みダイを規定します。つまり、上の画像のAおよびBの値です。最初のパラメーター（この例では8.5mm ）は、完全折り畳みの際の、パーツの中央線からの距離を規定します。2番目のパラメーター（ここでは95.5mm ）は完全折り畳みの際のパーツとダイ底部までの垂直距離を規定します。

EV - 完全折り畳みパンチ

これらは偏向なしの完全折り畳みに使うEV - Fパンチです。このようになパンチは　I0 HemPunchと呼ばれます。

このようなパンチには、さらに以下の入力が必要です。

  • テキスト FLAVOR = I 0 HemPunch は、これがI 0 HemPunchなことを示します。

  • テキスト PARAMS =-8,16,57.5 は、正しいシミュレーションに必要ないくつかの指標を提供します。

完全折り畳みプロセスを規定する3つのパラメーターがあります。つまり、上の画像のA、B、Cの値です。これらのパラメーターは、HemPunchで規定されているものと同じです。

ロールベンドダイ

ロールベンドダイはウィングダイと呼ばれます。

このツールには、さらに以下の入力が必要です。

  • テキスト FLAVOR = WingDie は、これがウィングツールなことを示します。

  • テキスト PARAMS = 10.49,100 はウィングの正しいシミュレーションに必要な指標を示します。

２つのパラメーターでウイングベンドピースの動きを規定します。最初のパラメーターは、ダイ中央線からウィング一つの回転の心合わせ点までの水平距離です。これは上の画像のAです。2番目のパラメーターは、ダイ底部かウィング回転の心合わせ点までの垂直距離です。これは上の画像のBです。通常、Bと全高は同じです。

セルフヘムダイ

これらは、下の画像のように、別にヘミングテーブルも備えた V ダイです。 このダイには、DXFファイルに以下のテキストエントリがあります： FLAVOR = SelfHemDie

さらに、2つの値のある PARAMS 入力が必要です。これら2つのパラメーターは、I 0 HemDieのパラメーターと同じです。

これらは、上の画像のAとB値です。最初のパラメーターは、完全折り畳み中のパーツの中央線からの距離を規定します。2番目のパラメーターは、完全折り畳み中のパーツとダイ底部の垂直距離を規定します。

ダブルVダイ

複数の V 幅があり、他の V を使用するために IAxis が必要なダイは、ダブル V ダイと呼ばれます。いくつかのルールに従って下さい：

  • FLAVOR=DoubleV 設定でダイの dxfs を各 V 幅ごとに 1 つ作成します。

  • 両方のダイに同じ ORDERNO を設定します（空でなければ任意の文字列にできます）

  • YSHIFT 値を正しいIAxis値に設定して使用します

  • ダイの底で、Vの下の中央にポイントマーカーを引っ張り、成形ジオメトリーがどこにあるかを示します

ばね式のダイ（フェザーヘミングダイス）

ばね式のダイには、DXFにこのようなテキス入力をして下さい： FLAVOR = SpringHem

さらに、次の追加入力が必要です：

  • テキスト PARAMS =-2,30,10 は、正しいシミュレーションに必要な指標を提供します。

パンチアダプタ

ツールインポーターは、_パンチアダプタ_をインポートできます。フラックスが描画を正しくパンチアダプタと認識できるように、ファイルには以下のテキスト入力が必要です。

  • UPPER=PTrumpf のようなテキストは上部の取り付けタイプを示します (通常、アダプタをトルンプ（社） マシンに取り付ける場合は PTrumpf になります)。

  • LOWER = PEHT のようなテキストは、下部取り付けタイプを示します（これは アダプタに取り付けられるパンチのタイプです）。

  • HEIGHT=64 のようなテキスト。フラックス はアダプタの高さを自動的に推測できないため、これが必要です。

ブレードベースサポート

ツールインポーターは、_ブレードベースサポート_をインポート出来ます。一部のブレードベースサポート (OW209/S に類似したものなど) は、裾を平らにするために使用する完全折り畳みパンチでもあります。このようなブレードベースサポートには、いくつかの追加データが必要です

必須パラメーター UPPER、LOWER、HEIGHT については、上記の「パンチアダプター」に関するセクションで説明されており、そこでの意味と同じです。さらに、フラックスがベースサポートを完全折り畳みツールとして使用するには、次の追加入力が必要です：

  • テキスト SPLFLAVOR= HemPunch は、アダプターが完全折り畳みツールなことを示します。

  • テキスト PARAMS=-6,18,60 は、完全折り畳みプロセスの正しいシミュレーションに必要な特定の指標を提供します。

完全折り畳みプロセスを規定する3つのパラメーターがあります。上の画像のA、B、Cの値です。最初のもの（この例では-6 mm ）は、パンチの平らな垂直面が曲げ線からどのくらい後ろにあるかを規定します。この場合、平らな垂直面は曲げ線の前にあるため、値は-6です。2番目のパラメーター（ここでは18 mm ）は、完全折り畳みの端が曲げ線のどのくらい前になるかを規定します。最後のパラメーター（ここでは60 mm ）は、パンチ端から完全折り畳みの端までの垂直距離です。

半径ベースサポート

これは、半径シャフトツール用ベースサポートです。

次の追加入力が必要です：

  • テキスト FLAVOR = RHolder は、アダプタが半径シャフトベースホルダーなことを示します。

  • テキスト PARAMS = 100 は、NCプログラムが出力するツールの高さです。

  • このツールの高さは、原点からベースサポートの逆Vまでの距離です。これは下の画像のAです。

ダブルVベースサポート

ダイを2個保持できるダイベースサポートです。

次の追加入力が必要です：

  • テキスト FLAVOR = DVHolder は、アダプタがDVHolderなことを示します。

  • テキスト PARAMS=0,36.2 は、ダイをベースサポートの別のスロットへ移動する数値を示します。

このツールを規定するには、2つのパラメーターが必要です。最初の値（この例では0mm）は、ベースサポートの原点から最初のスロットがどれだけ離れているかを示します。この場合、最初のスロットが原点に一致しているため、値は0です。 2番目のパラメーター（ここでは36.2mm ）は上の画像ではAで、ベースサポートの原点から2番目のスロットまでの距離を示します。
