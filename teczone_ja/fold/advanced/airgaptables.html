カスタムエアギャップテーブル

カスタムエアギャップテーブル

パネルベンダー用の エアギャップテーブル は、特定の板厚に対して加工時に使用される規格のエアギャップを対応付けたものです。これは各パネルベンダーに組み込まれていますが、JSON ファイル形式のカスタムエアギャップテーブルで上書き出来ます。 この JSON ファイルは air-gap-tables.json と呼ばれ、C:\ProgramData\Trumpf{APPNAME}\ Data\ System にあります。以下は典型的な air-gap-tables.json ファイルです。

{
  "version": 1,
  "airGapTables": [
    {
      "machine": "112002,112003",
      "material": "DC01",
      "values": [
        { "t": 0.75, "gap": 2.5 },
        { "t": 0.95, "gap": 3 },
        { "t": 1.25, "gap": 3.5 },
        { "t": 1.45, "gap": 4.5 },
        { "t": 1.65, "gap": 5.5 },
        { "t": 1.75, "gap": 6 },
        { "t": 2.05, "gap": 6.5 },
        { "t": 2.15, "gap": 7.5 },
        { "t": 2.55, "gap": 8 },
        { "t": 9.99, "gap": 8.5 }
      ]
    },
    {
      "machine": "112003",
      "material": "*",
      "values": [
        { "t": 0.75, "gap": 2.5 },
        { "t": 0.95, "gap": 3 },
        { "t": 1.25, "gap": 3.5 },
        { "t": 1.45, "gap": 4.5 },
        { "t": 1.65, "gap": 5.5 },
        { "t": 1.75, "gap": 6 },
        { "t": 2.05, "gap": 6.5 },
        { "t": 2.15, "gap": 7.5 },
        { "t": 2.55, "gap": 8 },
        { "t": 9.99, "gap": 8.5 }
      ]
    },
    {
      "machine": "*",
      "material": "Alluminio",
      "values": [
        { "t": 0.75, "gap": 2.5 },
        { "t": 0.95, "gap": 3 },
        { "t": 1.25, "gap": 3.5 },
        { "t": 1.45, "gap": 4.5 },
        { "t": 1.65, "gap": 5.5 },
        { "t": 1.75, "gap": 6 },
        { "t": 2.05, "gap": 6.5 },
        { "t": 2.15, "gap": 7.5 },
        { "t": 2.55, "gap": 8 },
        { "t": 9.99, "gap": 8.5 }
      ]
    }
  ]
}

  • ファイルにはエアギャップ値のコレクションがあります。各テーブルには、セレクターとして機能する マシンキー と 素材キー があります。

  • これらのキーは、ワイルドカード（すべてに一致）を示す ＊ (アスタリスク) に設定するか、または複数の値をカンマ区切りで列挙したリストに設定します。

  • 上記の例では 3 つのテーブルがあります。最初は機械番号 112002 または 112003 かつ素材 DC01 の場合に使用。2 つ目は機械 112003 で素材任意。3 つ目は素材 Alluminio で機械任意。複数一致する場合は最初に一致したテーブルが使用されます。

  • 各テーブルには、厚さキー (t) とそれに対応する エアギャップ値 (gap) の単純昇順リストです。厚さがその値以下の場合、そのギャップを使用します。一致テーブルがない場合は機械の初期設定テーブルを使用します。

air-gap-tables.json を変更した後は TecZone Fold を再起動してください。
JSON 形式が不正な場合、エラー表示後に初期設定テーブルへフォールバックします。
