A interface do usuário

A interface do usuário

TecZone Bend usa um design de interface do usuário inovador, de codinome Clarity. Quando você está trabalhando no TecZone Bend, você é apresentado a uma visão organizada e minimalista.

Na interface do TecZone Bend, você não encontrará:

  • Muitas linhas de ícones empilhados, com apenas um pequeno número sendo relevante para o contexto atual.

  • Estruturas de menu profundas com dezenas de menus.

  • Os menus da interface do usuário ocupam muito espaço na parte superior da tela.

Em vez disso, a interface Clarity funciona com base no princípio de Select, em seguida, Operar.

  • Selecione as coisas clicando nelas.

  • Um painel do Clarity aparece, contendo diversas Settings e Operações para a coisa selecionada.

  • O Painel geralmente contém alguns controles para Estender a seleção selecionando outras coisas semelhantes.

  • Muitas vezes, o painel também contém controles para Navegação pelas diversas coisas com as quais você está trabalhando.

Esses princípios de interface do usuário são consistentes em todos os modos de trabalho do TecZone Bend. Se você está limpando um modelo 3D, organizando peças em uma chapa aninhada ou editando ferramental de prensa-dobradeira, você usará o mesmo conjunto de conceitos-chave. Para tornar essas ideias abstratas mais concretas, vamos fazer uma pequena excursão e ver a interface Clarity em ação (vamos pegar exemplos do relatório TecZone Bend e do TecZone Bend para ilustrar).

Visita guiada

Este é um tour rápido sobre como a interface do Clarity funciona. Há apenas um um punhado de conceitos simples que são repetidos em todos os lugares no TecZone Bend, e entender esses conceitos significa que a maior parte do TecZone Bend pode ser usada sem muita leitura ou treinamento. Em outras palavras, se você ler esta seção, poderá pular a leitura do resto do guia do usuário do TecZone Bend!

Um relatório de amostra

Você pode aprender mais sobre como criar e editar modelos de relatório em uma seção posterior. Por enquanto, vamos apresentar alguma terminologia de relatório para que possamos usá-la para ilustrar conceitos da interface do Clarity.

A imagem acima mostra alguns dos tipos de entidades que podem ser colocadas em um relatório.

  • Um Label é um texto simples; você pode definir várias propriedades, como a fonte, alinhamento de texto e a cor.

  • Um Field exibe alguns dados reais coletados da Peça subjacente.

  • Uma Shape é uma geometria simples (como uma elipse, retângulo ou linha) que pode ser usada para adicionar fundos, destaques ou divisores em um relatório.

  • Uma Table exibe dados da Peça subjacente que tem uma forma tabular (como uma planilha de informações). Neste exemplo, podemos ver uma lista de operações de dobra desta peça, e alguns atributos são exibidos para cada uma delas.

Com esse conhecimento básico sobre entidades de relatório, podemos agora começar a explorar a interface do Clarity.

Princípio: Clique em algo para editar

Clique em um campo e será exibido um Painel Clarity (nesta discussão, vamos simplesmente chamá-lo de painel). Aqui está um diagrama anotado sobre como ele poderia parecer:

Clicar em um campo o seleciona e exibe um Painel para editá-lo. Aqui estão alguns elementos que vemos neste painel:

  • A barra de legendas pode ser usada para reposicionar o painel (você pode clicar em em qualquer lugar no fundo do painel e arrastá-lo). Se você definir o pino como inserido, então os painéis sempre abrem nesse local na tela. Caso contrário, os painéis abrem perto da localização do cursor do mouse.

  • Caixas de edição, caixas de entrada de números, listas de opções, seletores de fonte e alinhamento são todas as diversas maneiras pelas quais as configurações do campo são visualizadas e editadas.

  • Os botões de ação são como diversas operações são executadas no objeto selecionado. Nesse caso, pode-se excluir o campo do relatório clicando no botão Delete.

  • Alguns painéis podem conter muitos elementos. Para reduzir a desordem, os painéis são divididos em seções usando Expansores. Na imagem acima, você pode ver que o expansor Visual está fechado (ocultando todas as configurações nessa seção), enquanto o expansor de Fonte está aberto. Você pode clicar em qualquer lugar no expansor para abrir ou fechar uma seção.

Princípio: Selecione múltiplas coisas para editá-las juntas

Você pode selecionar entidades adicionais mantendo pressionada a tecla Shift ao clicar nelas. Quando múltiplas entidades são selecionadas, um painel é exibido que permite editar as configurações de todos eles ao mesmo tempo.

Na imagem acima, temos 3 campos que selecionamos usando o mecanismo Shift+Click. O painel que é exibido nos permite editar as configurações de todos estes três campos ao mesmo tempo. (Aqui temos o expansor Visual aberto, e o expansor Font fechado). Depois de editar a cor de fundo e as configurações de largura da caneta, veja como fica o relatório:

As alterações que fizemos são aplicadas a todos os itens selecionados. Neste cenário de edição múltipla, observe que a configuração Expression não é exibida; esta é uma das configurações que é improvável que queiramos editar junto para um conjunto de campos (não gostaríamos que 3 campos exibissem o mesmo valor da Peça). Ajustes que normalmente teriam valores distintos para cada entidade não são exibidos nesses painéis de edição de grupo.

Se selecionarmos um Field e uma Shape ao mesmo tempo, este é o painel que é exibido:

  • Agora a barra de legenda diz 2 Itens de relatório pois estamos editando itens de relatório, mas não do mesmo tipo.

  • Apenas as configurações comuns entre Field e Shape são exibidas (neste caso, essas são apenas as configurações na seção Visual).

Princípio: A seleção múltipla pode fornecer operações adicionais

Além do subconjunto comum de configurações e operações disponíveis em múltiplos objetos, a seleção múltipla pode, às vezes, apresentar novas operações ou configurações que não estão disponíveis em objetos individuais. Tomemos um exemplo da vista de dobra para ilustrar isso. Na visualização de dobra, clicar em uma dobra exibe um painel para editar as configurações da dobra e realizar algumas operações nele:

A imagem acima mostra o painel para a dobra 3. Vamos agora segurar Shift pressionada e selecionar a dobra 4 também. O painel que é exibido agora tem a seguinte aparência:

Existem algumas configurações que podemos editar em comum (a configuração de pular dobra ou a configuração do número da estação). No entanto, vemos um novo botão aqui, chamado Group. Quando duas dobras são selecionadas, e estas são colineares, elas podem ser agrupados em uma única operação de dobra. Isso é detectado automaticamente, e esse novo botão Group é exibido; esta operação não está disponível quando apenas uma dobra única é selecionada. Clicar nesse botão agrupa as duas dobras, como você pode ver na imagem abaixo.

Agora, existem apenas 3 operações de dobra nesta parte. Além disso, a dobra 3 agora é uma dobra agrupada e você vê um novo botão aparecer: Ungroup.

Princípio: Use o subpainel de seleção para modificar a seleção

Um painel Clarity geralmente terá um subpainel *de íons*Select perto da parte inferior. Este painel fornece alguns botões para modificar a seleção de diversas formas. Normalmente, há botões para selecionar mais objetos que são similares de uma forma ou de outra. Por exemplo, algumas formas para expandir a seleção seriam:

  • Selecionar todos os campos em um relatório.

  • Selecionar todas as polilinhas na mesma camada.

  • Selecionar todos os acertos de puncionamento em uma peça que usem a mesma ferramenta de puncionamento.

  • Selecionar todas as dobras que usam o mesmo ajuste.

Aqui está um exemplo do editor de relatórios de um painel de seleção sendo exibido quando clicamos em um Campo em um relatório:

O subpainel de seleção tem um fundo estampado distinto e botões arredondados. Neste exemplo, você pode ver os botões para selecionar Todos os Campos, selecione Todos os Campos de Dobra (campos relacionados aos dados CAM de dobra) e Toggle (o conjunto de seleção atual).

A lista real de botões de seleção exibidos neste subpainel irá variar dependendo do contexto. Você descobrirá que os cenários mais comuns são tratados muito convenientemente por meio desses botões de seleção.

Princípio: Use o subpainel de navegação para se mover por toda a árvore de entidade

O conjunto de entidades em uma peça, relatório ou layout geralmente possui uma estrutura hierárquica. Chamamos isso de Árvore da entidade. A estrutura é normalmente representada como um árvore de cabeça para baixo, com o objeto 'pai' no topo e os filhos abaixo dele. Este é um exemplo de uma árvore de entidade em um relatório.

  • Um relatório pode conter múltiplos itens. Alguns destes itens podem ser tabelas.

  • Uma tabela de um relatório pode ter múltiplas colunas.

Assim, existem 3 níveis nesta árvore, e uma representação gráfica desta árvore pode ser a seguinte:

Este relatório possui 2 Rótulos, 1 Campo e 1 Tabela. A tabela tem 4 colunas. Quando clicamos na tabela, vemos um painel como este:

O painel de navegação fornece botões de navegação direcional para percorrer a árvore de entidades. Assim como o painel de seleção, o subpainel de navegação possui um plano de fundo texturizado e aparece na parte inferior do painel, mas logo acima do subpainel de seleção.

Os botões Prev. e Next se movem entre os itens no mesmo nível da árvore de entidades. Assim, clicar nesses botões selecionaria os outros itens no relatório, como o Label ou os itens de Field no relatório.

Clicar no Report move para cima da tabela para o nível superior - o próprio Relatório. O painel que é exibido quando você clica no botão Report mostra algumas configurações relacionadas ao relatório em si, como o tamanho do papel e as margens:

Agora estamos editando o relatório; estamos na raiz da árvore de entidades. Não há direção para ir exceto para baixo, portanto, o subpainel de navegação tem somente um botão ( Itens) para descer até a lista de itens. Clicar nele seleciona a tabela novamente.

A partir da tabela, o botão de navegação para baixo Columns botão nos move para baixo um nível e nos coloca no modo de edição de coluna. Isso é o que acontece quando clicamos no botão de navegação Columns do painel da tabela:

As configurações para a coluna selecionada na tabela são exibidas. Clicar nos botões Prev. e Next agora move a seleção entre as colunas da tabela, já que esse é o nível da árvore com a qual estamos trabalhando. Não há um botão de navegação para baixo, pois este é o nível mais baixo da árvore. O botão Table sobe um nível, e estamos de volta para editar a tabela inteira.

Resumo

  • Clique em algo para editar.

  • Selecione múltiplas coisas e você poderá editá-las juntas.

  • Você também pode editar entidades de diferentes tipos juntas.

  • A seleção múltipla pode fornecer configurações e operações adicionais.

  • Use o subpainel Seleção para modificar a seleção de forma inteligente.

  • Use o subpainel Navegação para percorrer a árvore de entidades.
